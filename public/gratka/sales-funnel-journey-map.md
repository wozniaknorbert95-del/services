## How the journey is structured

Every step keeps two things visible: a **progress bar** and a **sticky cart** with a running total. Your client always knows where they are and what they're building toward.

| Phase | What the client does | What you get |
|---|---|---|
| **Welcome** | Orient — what they'll configure | Fewer drop-offs from confusion |
| **01 Foundation** | Pick core package / branding base | Structured starting point |
| **02 Vehicle** | Add mobile branding options | Upsell without a sales call |
| **03 Workwear** | Choose clothing branding | Clean option data |
| **04 First impression** | Select print / physical materials | No "what did you want again?" emails |
| **05 Visibility** | On-site presence options | Same capture format every time |
| **06 Tools** | Equipment branding choices | Logic rules applied automatically |
| **07 Premium** | Finishing touches | Final upsell in flow, not in inbox |
| **Checkout** | See open price · upload assets · pay or book | Qualified lead with full spec |

## Built-in guardrails

- **Open pricing** — client sees the number before committing, not after three email rounds.
- **Option logic** — minimum values, dependencies and rules enforced in the flow.
- **Single path** — wizard-only; no parallel quote channels that bypass the system.
- **Structured output** — every enquiry captured the same way.
