**Version:** 3.0  
**Status:** Canonical — post-audit positioning, message hierarchy, price integrity and route copy rules  
**Language:** English for public-facing site; Polish allowed only in internal notes.

---

## 1. Core decision

Every page must help the buyer answer:

> **Can Norbert design and deploy a system that improves my business?**

The buyer must not leave thinking only:

> “He builds websites / chatbots / automations.”

**Conversion Systems Architect**

**Lead qualification and automation systems for Dutch small businesses.**

### Anti-positioning

> Not a web designer. Not an AI chatbot builder. I design the conversion system that turns enquiries, quotes and admin into a controlled workflow.

## 9. Solution positioning

### Inbox Killer

**Problem:** important emails, leads, invoices and noise are mixed together.  
**System:** classify, draft, queue and human-approve replies.  
**Effect:** fewer missed leads, fewer inbox hours, no autonomous sending.

### Web Upgrade

**Problem:** the website looks outdated or fails to convert mobile visitors.  
**System:** modern conversion site with clear paths, lead capture, proof and analytics.  
**Effect:** more trust and cleaner enquiries without pretending it is a full automation ecosystem.

### Sales Funnel / Wizard Cash Engine

**Problem:** quoting and booking require repeated manual questions.  
**System:** guided configurator, structured qualification, price/quote logic, checkout or handoff.  
**Effect:** serious leads self-qualify before the owner spends time.

### Lead Magnet Game

**Problem:** cold traffic ignores static forms.  
**System:** interactive/gamified acquisition flow with reward ladder and wizard handoff.  
**Effect:** longer engagement and better-qualified contacts.

### Managed Automation

**Problem:** systems decay after launch.  
**System:** monitoring, tuning, small improvements and periodic review.  
**Effect:** the system stays useful without locking the client in.

## 10. Proof standards

Allowed:

- real screenshots,
- diagrams,
- process artefacts,
- verifiable counts,
- labelled pilot/demo material,
- anonymised delivery where status is explicit.

Forbidden:

- fake client logos,
- fake testimonials,
- unlabelled demo fixtures,
- unfinished video placeholders on public money pages,
- public pilot case pages with only a title and placeholder.

### Status labels

Use consistently:

| Label | Meaning |
|---|---|
| LIVE | running in production |
| PARTIAL | useful parts live; planned parts named clearly |
| PILOT | flow exists but not full production proof |
| DEMO | fixture or illustrative asset |
| PLANNED | not live; must not be sold as live |

## 11. Founder voice

Use first person for founder credibility:

> I built this first to run my own business. Quietforge is how I deploy the same architecture for other small businesses.

Avoid:

- “agency” as self-description,
- anonymous “we” in founder story,
- video placeholders that read unfinished.

Use “we” only for delivery partnership:

> We map your process. I design the system. You approve what ships.
