## Zasady profilu (strategia, nie UI checklist)

### P1 — Tożsamość

| Zasada | Szczegół |
|--------|----------|
| Primary label | Conversion Systems Architect ([marketing-rules](../../canons/marketing-rules.md) MR-01) |
| Company | Quietforge |
| FlexGrafik | W About / Experience / proof — nie w headline jako „drukarnia” |
| Banner | Zachować Quietforge + 8-repo diagram (audyt: 5/5) |

### P2 — Konwersja

| Zasada | Docelowy kierunek |
|--------|-------------------|
| Jedna akcja | Book Automation Map ([conversion-pipeline](../conversion-pipeline.md) L3) |
| Featured | quietforge home, book-discovery, 1–2 case studies |
| Link w poście | W **pierwszym komentarzu**, nie w body (zasięg) |
| Pricing | Jawne odsyłanie do [marketing-strategy §8](../marketing-strategy.md) — nie „contact for pricing” |

### P3 — Feed

| Zasada | Szczegół |
|--------|----------|
| Język postów | EN |
| Struktura | Problem → System → Effect → CTA (komentarz) |
| FlexGrafik w poście | Screenshot / metryka / „in my print business” — nie oferta druku |
| Founder story | Dozwolone 20% mix ([05-content-pillars](./05-content-pillars.md)) — bez funding ask |
| Investor | [08-investor-track](./08-investor-track.md) — nie na głównym feedzie teraz |

### P4 — Honesty (LIVE / PARTIAL / DEMO)

Zgodnie z [proof-rules.md](../../canons/proof-rules.md):

| Mów | Nie mów |
|-----|---------|
| „Wizard checkout LIVE on zzpackage” | „System 80% complete” |
| „Jadzia COI Phase A+B LIVE; Procurement Brain PLANNED” | „Almost finished, 2 months left” |
| „161 SKUs” (site SSoT) | „200+ SKUs” jeśli Services nadal ma starą liczbę |
| PROVEN / DEMO / PLANNED labels | Fake testimonials, fake logos |

## Szablon mentalny jednego posta

```
1. Hook — problem NL SMB (1–2 linie)
2. System — co zbudowałeś (bez repo dump)
3. Proof — 1 liczba lub screenshot + status LIVE/PARTIAL
4. Effect — co zmienia dla biznesu
5. CTA — w komentarzu: Book Automation Map + URL + UTM
```
